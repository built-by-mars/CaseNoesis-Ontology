# CQ-27 through CQ-36 disposition — literal same-bundle SHACL + coverage

**Date:** 2026-07-12  
**Scope:** `mcp_server/validation_bundle.py`, `mcp_server/concept_coverage.py`, `mcp_server/graph_validator.py`, tests  
**Do not tag / do not push** (per directive).

## Summary

SHACL and strict concept coverage now share one `ResolvedValidationBundle` (exact resources + fingerprint), not merely matching profile name lists. Upper-ontology terms fail closed unless profiles are explicit. Caches are content-addressed and lock-guarded; manifests are portable and atomically written; validation stages report independently.

## CQ disposition

| CQ | Status | Implementation |
|----|--------|----------------|
| **CQ-27** | Done | `check_graph_concepts(..., bundle=ResolvedValidationBundle)` loads declared terms from that bundle’s resources and records `bundle_fingerprint`. `validate_graph_file` constructs one bundle when `profiles` or (`extensions` + `strict_concepts`) and passes it through. |
| **CQ-28** | Done | Bundle resolution uses the same subset/full mode as SHACL (`validation-subset.json` vs full manifest). Coverage accepts only terms from those exact files. Direct `extensions=` (no bundle) also prefers subset mode. |
| **CQ-29** | Done | `profiles=None` / empty ⇒ **zero** upper profiles authorized. Upper-namespace terms → `profile_not_selected`. Documented in `concept_coverage` module docstring and `PROFILE_NOT_SELECTED_GUIDANCE`. Recipe gate always passes `profiles=[...]`. |
| **CQ-30** | Done | Dedicated `profile_not_selected` field: `{iri, required_profile, used_as, selected_profiles}` on `ConceptCoverageReport` / MCP payload. No longer embedded in `undeclared_classes` strings. |
| **CQ-31** | Done | Ontology parse failures are collected as `OntologyParseError` (path, role, error, profile_id, extension) and fail closed (`verification_status=could_not_verify`). |
| **CQ-32** | Done | Bundle + declared-term + upper-registry caches use content SHA-256 fingerprints and `threading.RLock`. |
| **CQ-33** | Done | Canonical cache keys normalize aliases and sort set-like inputs. `cache_status` is `hit` / `miss` / `stale` / `disabled`. |
| **CQ-34** | Done | Profile `depends_on` expands with cycle errors (`profile_dependency_cycle`); `incompatible_with` enforced; inference combined deterministically (subset skip only when a validation-subset is actually used); `requested_profiles` vs normalized `profiles` retained. |
| **CQ-35** | Done | Portable manifests omit absolute paths; include `resolver_schema_version`; `write_manifest` is atomic (temp + `replace`). |
| **CQ-36** | Done | Independent stages: `bundle_resolution`, `shacl_execution`, `shacl_conformance`, `coverage_execution`, `coverage_conformance`; report includes `validator_version`, fingerprint, resources, cache status. |

## Behavioral note (callers)

Graphs that use BFO / gUFO / PROV-O / Time / GeoSPARQL / FOAF / ORG / PROF terms **must** pass `profiles=[...]` (or go through recipe validation which already does). Omitting profiles no longer silently accepts the full upper-ontology registry.
